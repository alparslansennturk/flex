# Mayıs 2026 — Fix Detayları

### FIX-217: Kişisel şablon ikinci kez kullanılamıyordu — busyGroupIds fix
**Sorun:** `AssignActivateModal`, ilk kullanımda oluşturulan `tasks` dokümanını `templateId` üzerinden bulup grubu "busy" sayıyordu. İkinci açılışta tüm gruplar seçilemez hale geliyor, "Aktife Al" butonu disabled kalıyordu.
**Düzeltme:** `AssignActivateModal`'a `templateScope?: string` prop'u eklendi. `scope === "personal"` ise `busyGroupIds` boş bırakılır (busy sorgusu atlanır). `AssignmentLibrary`'de modal çağrısına `templateScope={assignModalTask.scope}` eklendi.
**Dosyalar:** `AssignActivateModal.tsx`, `AssignmentLibrary.tsx`

### FIX-218: Kişisel şablon `addDoc` — description undefined Firestore hatası
**Sorun:** `AssignmentLibrary` `onConfirm` içinde `description: t.description` yazılıyordu. `QuickAssignModal` şablon oluştururken `description` alanı set etmediğinden `t.description === undefined` geliyordu; Firestore `undefined` değeri reddetti (`FirebaseError: Unsupported field value: undefined`).
**Düzeltme:** `description: t.description ?? null` — undefined yerine null yazılır.
**Dosyalar:** `AssignmentLibrary.tsx`

### FIX-220: Grading "Bekleyen" boş görünüyordu — composite index + published + yerel tarih
**Sorun:** `GradingTabs` yalnızca `status=completed` sorgusu yapıyordu. Deadline geçmiş `active` ödevler hiç çekilmiyordu. Sonraki denemede `status + ownedBy` çift `where()` kullanıldı; Firestore composite index olmadığında `catch` olmadığı için sessizce hata verdi.
**Düzeltme:** Tek alan sorgusu (`ownedBy==uid`, `createdBy==uid`) paralel çalıştırılıp client-side dedup ve status filtresi uygulandı. `published` statüsü eklendi. `todayStr` UTC→yerel saat (`getDate()` bazlı). Karşılaştırma `endDate < todayStr` (gün bitmeden bekleyene düşmez).
**Dosyalar:** `src/app/dashboard/grading/page.tsx`
